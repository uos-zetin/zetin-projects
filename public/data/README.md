# 프로젝트 내용·사진 편집 가이드 (콘텐츠 담당자용)

이 폴더(`public/data/`)가 **사이트의 모든 내용**입니다. 코드는 건드릴 필요 없이 여기 두 가지만 다루면 됩니다.

- `projects.json` — 프로젝트 목록(제목·설명·부원·기술 등)
- `images/` — 프로젝트 사진 파일

> 💡 더 쉬운 방법: **관리자 페이지(`/admin`)**에서 ZETIN 계정으로 로그인하면 폼으로 추가·수정·삭제·순서변경·이미지/링크 편집을 할 수 있습니다. 이 문서는 `projects.json`을 **직접 편집**할 때의 안내입니다.

## 1. 프로젝트 내용 편집 — `projects.json`

`projects.json`을 열면 `"projects": [ ... ]` 안에 프로젝트가 하나씩 `{ ... }`로 들어 있습니다.
하나를 복사해 붙여넣고 값만 바꾸면 새 프로젝트가 추가됩니다.

각 항목의 필드:

| 필드 | 필수 | 설명 |
|---|---|---|
| `id` | ✅ | 영문/숫자 고유 식별자(겹치면 안 됨). 예: `"robocup-humanoid"` |
| `title` | ✅ | 프로젝트 이름 |
| `category` | ✅ | 분류. 같은 값끼리 필터 묶임. 예: `"휴머노이드"`, `"드론"` |
| `year` | ✅ | 연도(숫자). 예: `2025` |
| `summary` | | 카드에 보이는 한두 줄 요약 |
| `description` | | 상세 페이지의 "프로젝트 개요" 본문. **마크다운 문법 지원**(굵게 `**...**`, 목록 `- `, 링크 `[글](url)`, 표, 코드 `` `...` `` 등). 줄바꿈은 `\n` 그대로 반영됨 |
| `members` | | 참여 부원 이름 배열. 예: `["홍길동", "김철수"]` |
| `tech` | | 사용 기술 태그 배열. 예: `["ROS", "강화학습", "GPU"]` |
| `awards` | | 수상 내역(있을 때만) |
| `links` | | 버튼으로 표시될 링크 배열. 예: `[{ "label": "원본 글 보기", "url": "https://..." }]` (라벨/주소에 "github"가 들어가면 강조 버튼) |
| `thumbnail` | | 카드·상세 대표 사진 경로(아래 2번 참고). 없으면 회색 placeholder 표시 |
| `images` | | 상세 "제작 사진"에 들어갈 사진 경로 배열. 없으면 placeholder 표시 |
| `featured` | | `true`면 카드에 `FEATURED` 뱃지 |

> 표시 순서는 **JSON 배열에 적힌 순서**입니다(관리자 페이지에서 드래그로 변경 가능). 카테고리/연도/검색 필터는 데이터에서 자동 생성됩니다.

## 2. 사진 넣기 — `images/` 폴더

1. 사진 파일을 `public/data/images/` 안에 넣습니다. (예: `robocup.jpg`)
2. `projects.json`에서 그 프로젝트의 `thumbnail`/`images`에 경로를 적습니다:

```json
{
  "id": "robocup-humanoid",
  "title": "로보컵 휴머노이드 (2족보행)",
  "thumbnail": "data/images/robocup.jpg",
  "images": ["data/images/robocup-1.jpg", "data/images/robocup-2.jpg"]
}
```

- 경로는 `data/images/파일명` 형식(폴더 기준 상대경로).
- 외부 주소(`https://...`)를 그대로 넣어도 됩니다.
- 사진을 안 넣으면 자동으로 회색 placeholder가 나옵니다(깨지지 않음).
- 권장: 카드용은 가로형(4:3), 대표/상세 상단은 16:9. 용량은 장당 수백 KB~1MB 이내로 줄여서.

## 3. 편집한 내용 확인하기

- **로컬 개발 중**: `npm run dev`로 띄워두면 `projects.json`/사진을 저장하는 즉시 화면에 반영됩니다.
- **배포 후(서버)**: 이 `data/` 폴더가 서버에 볼륨으로 연결되어 있어, 파일만 고치고 새로고침하면 반영됩니다(재빌드 불필요).

## 4. JSON 작성 시 주의

- 항목 사이, 필드 사이는 **쉼표(`,`)**로 구분. **마지막 항목 뒤에는 쉼표 없음**.
- 모든 키와 문자열 값은 **큰따옴표** `"..."`. 숫자(`year`)·`true`/`false`는 따옴표 없음.
- 저장 전 [jsonlint.com](https://jsonlint.com) 등에 붙여 넣어 문법 오류가 없는지 확인하면 안전합니다.
